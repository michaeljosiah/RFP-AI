# E2E chat verification

## Why

End-to-end verification of the chat flow is needed to confirm that user messages travel through the full stack — client, transport, and model backend — and return correct, well-formed responses before release. Without an automated E2E check, regressions in the chat pipeline can reach users undetected, since unit and integration tests do not exercise the complete request/response path.
