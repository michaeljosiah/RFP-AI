RFP Question Extractor LibreOffice + Microsoft Agent Framework

A .NET console application that extracts every question from Word, PDF and Excel questionnaires into two linked JSON documents — a structure schema and a flat answerable question list. LLM calls go through Microsoft Agent Framework (model-agnostic, structured output). Rendering and document parsing use a fully open-source, self-hosted stack (Gotenberg/LibreOffice + Docnet.Core + Open XML SDK + ClosedXML).

Same Core & contract as the Telerik edition. The domain model, the two JSON schemas, the four prompts, the pipelines (document + Excel), reconciliation, the Microsoft Agent Framework LLM layer, and the LLM-tool design are identical to spec-telerik.html. Only §10 (engine adapters), §13 (prereqs) and §15 (licensing) differ. One stable contract, two interchangeable engines.
Implementation status. Built and verified. This OSS engine ships as RfpExtractor.LibreOffice in the same solution and is selectable at runtime with --engine=libreoffice. The full solution compiles with 0 warnings on .NET 10; the shared Core has 10 passing unit tests. Verified against the real M&G questionnaire (docx → 8.3k-char markdown via Open XML SDK + 1 ground-truth table + 3 watermark-free pages via Gotenberg→Docnet→SkiaSharp) and a test workbook (xlsx → 6 label + 6 empty answer cells with exact addresses via ClosedXML). The LLM legs compile against MAF 1.12.0 and need an IChatClient endpoint to run. Deviations found during the build are folded in below and flagged "verified by build".

1. Overview & Goals

Questionnaires arrive as Word (.docx), PDF, or Excel (.xlsx). We turn them into a concrete, answerable question list and record exactly where each answer belongs.

  • Word / PDF — dual pipeline: a vision leg (Gotenberg→PDF→Docnet.Core→PNG→agent) and a structured-text leg (Open XML SDK → Markdown → agent), then reconciliation.
  • Excel — grid-first: read cells directly with ClosedXML (exact addresses) → agent classifies labels vs answer cells; optional vision cross-check (xlsx→PDF→image via Gotenberg). Excel answers get exact Sheet!Cell bindings — no annotation needed.
  • Reconciliation — merge legs, tag found_by + confidence, route single-source items to a review queue.
  • Data residency — all rendering/parsing is self-hosted; only the LLM calls leave your network (front them through your approved Azure OpenAI / gateway).

Why MAF lives in Core. Microsoft.Agents.AI and Microsoft.Extensions.AI are provider-agnostic abstractions, so they belong in Core. The concrete model provider (Azure OpenAI shown) is an IChatClient constructed in the CLI. Swapping providers never touches Core.

4.3 The 1:1 invariant

Invariant. The answer_target set in document_schema equals the set referenced by questions. No orphans, no duplicates. Validate after each extraction and after reconciliation.
// RfpExtractor.Core/Validation/InvariantValidator.cs
public static class InvariantValidator {
    public static IReadOnlyList<string> Validate(ExtractionResult r) {
        var errors = new List<string>(); var schema = new HashSet<string>();
        foreach (var s in r.DocumentSchema.Sections)
        foreach (var it in s.Items) {
            if (it.Type == ItemType.Table && it.Table is not null)
                foreach (var c in it.Table.Cells) Add(schema, c.AnswerTarget, errors);
            else if (it.AnswerTarget is { } t) Add(schema, t, errors); }
        var q = new HashSet<string>();
        foreach (var it in r.Questions) {
            if (!q.Add(it.AnswerTarget)) errors.Add($"Duplicate question target: {it.AnswerTarget}");
            if (!schema.Contains(it.AnswerTarget)) errors.Add($"Question {it.QuestionId} -> missing target {it.AnswerTarget}"); }
        foreach (var t in schema) if (!q.Contains(t)) errors.Add($"Schema target {t} has no question");
        return errors;
        static void Add(HashSet<string> set, string t, List<string> e){ if(!set.Add(t)) e.Add($"Duplicate schema target: {t}"); } } }

5. Prompts

String constants in Prompts.cs; each becomes an agent's instructions (§9). Structured output forces the schema.

5.1 Vision

You are a precise document-extraction engine for RFP/questionnaire automation.
You are given ONE image of a single page. Return a single object with "document_schema"
and "questions" as structured output. Do not add commentary.

CORE
- Mint a unique "answer_target" ("AT-0001", ... in reading order) for EVERY answerable
  location: each open question, each document/data request line, each yes/no, and each
  fillable data-entry table cell.
- INVARIANT: every answer_target appears exactly once in document_schema AND exactly once
  in questions. No orphans, no duplicates.

RULES
1. VERBATIM in the schema; questions may NORMALISE phrasing but keep the original in
   "verbatim_source". Never answer anything.
2. "layout" tables (label column + question column) are NOT emitted as tables - extract
   content as items, use label cells as section names. "data_entry" grids ARE emitted:
   column_headers, row_headers, intro_verbatim, one cell per fillable cell (own
   answer_target,row,column,answer_type) AND one question per cell phrased standalone from
   context+column+row. e.g. "What was the firm's AUM 1 year ago?"
3. MULTI-PART sharing one box -> one question + "sub_questions"; separate boxes -> separate targets.
4. answer_type: text|long_text|number|currency|percentage|date|yes_no|document_upload.
5. SECTIONS from headings/bands; section+subsection+page; slug ids; reading order.
6. "truncated"/"unreadable" flags; exclude headers/footers/page-numbers/watermarks/legal;
   put document-identifying fields in metadata.

5.2 Structured text

You are a precise document-extraction engine. Input: the FULL document as Markdown (tables
preserved; headings as '#'). Return one object with "document_schema" and "questions".

[... identical CORE and RULES 1-6 as Vision ...]

TEXT MODE: markdown tables are literal - header row = column_headers; leftmost body column =
row header; every EMPTY body cell is a fillable data_entry cell (own answer_target +
question). A body cell with text is context, not an answer cell.

5.3 Reconciliation

You merge two extractions of the SAME questionnaire. Input JSON has "primary" (questions[])
and "secondary" (questions[]). Return one "questions" array as structured output.
1. Same if normalised meaning matches OR same cell (row+column under same section). Merge.
2. On merge PREFER primary's verbatim_source + answer_target + binding; pick cleaner
   question_text; union sub_questions.
3. found_by = both | (single source). confidence = high(both) | medium(single-source cell) |
   low. needs_review = (found_by != both). Never drop single-source questions.
4. Preserve fields; keep answer_target stable. Never invent questions.

5.4 Excel grid

You are a precise spreadsheet-extraction engine. Input: ONE worksheet as JSON
{ "sheet":<name>, "cells":[ {address,text} ... ] } where answer cells have text = "".
Return one object with "document_schema" and "questions".
1. Classify LABEL cells (question text, row/column headers, section banners) vs ANSWER cells
   (empty cells in the used range under a header and/or beside a row label).
2. For EACH answer cell emit a schema entry (data_entry cell when in a header x row grid,
   else open_question) with a unique answer_target AND one question. question_text is a
   fluent standalone question from section + column header + row label.
3. binding = { "kind":"cell", "sheet":<name>, "address":<A1> } on the question; set
   schema_ref.row/column to the label texts.
4. answer_type from header wording; verbatim_source = exact labels.
5. INVARIANT: one answer_target per answer cell, one question each. Ignore styling-only /
   out-of-range cells.

6. Document Pipeline (Word / PDF)

// RfpExtractor.Core/Pipeline/DocumentPipeline.cs
public enum Strategy { Vision, Text, Both }
public sealed record ExtractionOptions(Strategy Strategy = Strategy.Both, int Dpi = 200, int MaxParallel = 4);

public sealed class DocumentPipeline {
    private readonly IDocumentRenderer _renderer; private readonly IStructuredTextExtractor _text;
    private readonly ILlmExtractor _llm; private readonly IReconciler _reconciler;
    public DocumentPipeline(IDocumentRenderer r, IStructuredTextExtractor t, ILlmExtractor l, IReconciler rec)
    { _renderer=r; _text=t; _llm=l; _reconciler=rec; }
    public async Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        ExtractionResult vision = new();
        if (opts.Strategy is Strategy.Vision or Strategy.Both) {
            var pages = await _renderer.RenderToImagesAsync(path, opts.Dpi, ct);
            var sem = new SemaphoreSlim(opts.MaxParallel);
            var per = await Task.WhenAll(pages.Select(async p => { await sem.WaitAsync(ct);
                try { return await _llm.ExtractFromImageAsync(p, ct); } finally { sem.Release(); } }));
            vision = ResultMerger.StitchPages(per); }
        ExtractionResult text = new(); StructuredDocument? structured = null;
        if (opts.Strategy is Strategy.Text or Strategy.Both) {
            structured = await _text.ExtractAsync(path, ct);
            if (!string.IsNullOrWhiteSpace(structured.Markdown))
                text = await _llm.ExtractFromTextAsync(structured.Markdown, null, ct); }
        var gt = structured?.Tables ?? Array.Empty<TableStructure>();
        var result = opts.Strategy switch {
            Strategy.Vision => ReconciledResult.FromSingle(vision),
            Strategy.Text   => ReconciledResult.FromSingle(text),
            _               => _reconciler.Reconcile(primary: text, secondary: vision, gt) };
        foreach (var e in InvariantValidator.Validate(result.Merged)) result.Report.Warnings.Add(e);
        return result; } }
ResultMerger.StitchPages concatenates per-page schemas, renumbers AT-####/Q### continuously, and joins truncated:true items to the next page's first item. Mechanical loop.

7. Excel Pipeline

The grid leg is authoritative (exact cell addresses, no OCR); vision is an optional cross-check. Each sheet is a "page". Fill-back bindings are exact (Sheet!Cell), so Excel needs no annotation.

// RfpExtractor.Core/Pipeline/SpreadsheetPipeline.cs
public sealed class SpreadsheetPipeline {
    private readonly ISpreadsheetExtractor _grid; private readonly IDocumentRenderer _renderer;
    private readonly ILlmExtractor _llm; private readonly IReconciler _reconciler;
    public SpreadsheetPipeline(ISpreadsheetExtractor g, IDocumentRenderer r, ILlmExtractor l, IReconciler rec)
    { _grid=g; _renderer=r; _llm=l; _reconciler=rec; }
    public async Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        var wb = await _grid.ExtractAsync(path, ct);
        var perSheet = new List<ExtractionResult>();
        foreach (var sheet in wb.Sheets) {
            var payload = System.Text.Json.JsonSerializer.Serialize(new {
                sheet = sheet.Name, cells = sheet.Cells.Select(c => new { c.Address, c.Text }) }, Json.Options);
            perSheet.Add(await _llm.ExtractFromGridAsync(payload, ct)); }
        var gridResult = ResultMerger.StitchPages(perSheet.ToArray());
        if (opts.Strategy != Strategy.Both) {
            var single = ReconciledResult.FromSingle(gridResult);
            foreach (var e in InvariantValidator.Validate(single.Merged)) single.Report.Warnings.Add(e);
            return single; }
        var pages = await _renderer.RenderToImagesAsync(path, opts.Dpi, ct);        // xlsx -> pdf -> image via Gotenberg
        var per = await Task.WhenAll(pages.Select(p => _llm.ExtractFromImageAsync(p, ct)));
        var vision = ResultMerger.StitchPages(per);
        var result = _reconciler.Reconcile(primary: gridResult, secondary: vision, Array.Empty<TableStructure>());
        foreach (var e in InvariantValidator.Validate(result.Merged)) result.Report.Warnings.Add(e);
        return result; } }

// RfpExtractor.Core/Pipeline/PipelineRouter.cs
public sealed class PipelineRouter {
    private readonly DocumentPipeline _doc; private readonly SpreadsheetPipeline _sheet;
    public PipelineRouter(DocumentPipeline doc, SpreadsheetPipeline sheet) { _doc=doc; _sheet=sheet; }
    public Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        var ext = Path.GetExtension(path).ToLowerInvariant();
        return ext is ".xlsx" or ".xlsm" or ".xls" ? _sheet.RunAsync(path, opts, ct) : _doc.RunAsync(path, opts, ct); } }
Default for Excel is grid-only (Strategy.Text ⇒ grid). Add --strategy=both for the vision cross-check. Cap cells per agent call for very large sheets and log the truncation.

8. Reconciliation & Confidence

v2 — rebuilt after field testing (shared Core). Reconciliation now uses one-to-one ordered matching (same-leg repeats are distinct answer slots, never collapsed), verbatim-first keys (cells by row+column, then verbatim_source, then question_text), an optional LLM fuzzy pass (AgentFuzzyMatcher, default ON, --no-fuzzy), and post-merge target renumbering with secondary-only schema grafting so the 1:1 invariant holds. Full rationale: spec-telerik.html §8 — identical implementation for both engines.

// RfpExtractor.Core/Reconciliation/*.cs
public static class TextNormalizer {
    public static string Key(string s) => System.Text.RegularExpressions.Regex.Replace(
        (s ?? "").ToLowerInvariant().Trim(), @"[\s\p{P}]+", " ").Trim();
    public static string CellKey(SchemaRef r) => $"{r.SectionId}|{r.Row?.ToLowerInvariant()}|{r.Column?.ToLowerInvariant()}"; }

public sealed class Reconciler : IReconciler {
    public ReconciledResult Reconcile(ExtractionResult primary, ExtractionResult secondary, IReadOnlyList<TableStructure> groundTruth) {
        var merged = new Dictionary<string, Question>();
        var report = new ReconciliationReport { PrimaryCount = primary.Questions.Count, SecondaryCount = secondary.Questions.Count };
        void Ingest(IEnumerable<Question> qs, bool isPrimary) {
            foreach (var q in qs) {
                var key = q.Source == QuestionSource.TableCell ? "cell:" + TextNormalizer.CellKey(q.SchemaRef)
                                                               : "txt:"  + TextNormalizer.Key(q.QuestionText);
                if (merged.TryGetValue(key, out var e))
                    merged[key] = e with { FoundBy = FoundBy.Both, Confidence = Confidence.High, NeedsReview = false,
                        SubQuestions = e.SubQuestions.Union(q.SubQuestions).ToList() };
                else
                    merged[key] = q with { FoundBy = isPrimary ? FoundBy.Text : FoundBy.Vision,
                        Confidence = q.Source == QuestionSource.TableCell ? Confidence.Medium : Confidence.Low,
                        NeedsReview = true }; } }
        Ingest(primary.Questions, true); Ingest(secondary.Questions, false);
        var cellCount = merged.Values.Count(q => q.Source == QuestionSource.TableCell);
        var expected = groundTruth.Sum(t => Math.Max(0, (t.Rows - 1) * (t.Columns - 1)));
        if (expected > 0 && cellCount < expected)
            report.Warnings.Add($"Table cell undercount: found {cellCount}, grid implies ~{expected}.");
        var all = merged.Values.ToList();
        report.MergedCount = all.Count; report.AgreedCount = all.Count(q => q.FoundBy == FoundBy.Both);
        report.PrimaryOnlyCount = all.Count(q => q.FoundBy == FoundBy.Text);
        report.SecondaryOnlyCount = all.Count(q => q.FoundBy == FoundBy.Vision);
        var schema = primary.DocumentSchema.Sections.Count > 0 ? primary.DocumentSchema : secondary.DocumentSchema;
        return new ReconciledResult { Merged = new ExtractionResult { DocumentSchema = schema, Questions = all },
            ReviewQueue = all.Where(q => q.NeedsReview).ToList(), Report = report }; } }

9. LLM via Microsoft Agent Framework

All LLM work goes through MAF agents; Core references only the abstraction packages. One AgentLlmExtractor builds three agents (one per prompt) and uses structured output (RunAsync<ExtractionResult>). Vision is a DataContent image block. This class is identical to the Telerik edition — the LLM layer is engine-independent.

// RfpExtractor.Core/Llm/AgentLlmExtractor.cs
using Microsoft.Agents.AI; using Microsoft.Extensions.AI;
using RfpExtractor.Core.Abstractions; using RfpExtractor.Core.Models; using RfpExtractor.Core.Prompts;

public sealed class AgentLlmExtractor : ILlmExtractor {
    private readonly AIAgent _vision, _text, _grid;
    public AgentLlmExtractor(IChatClient chat) {
        // ChatClientAgentOptions has NO Instructions property (MAF 1.12) -> set it on ChatOptions.
        AIAgent Make(string name, string instructions) => new ChatClientAgent(chat,
            new ChatClientAgentOptions { Name = name,
                ChatOptions = new ChatOptions { Instructions = instructions, Temperature = 0f } });
        _vision = Make("vision-extractor", Prompts.Vision);
        _text   = Make("text-extractor",   Prompts.Text);
        _grid   = Make("grid-extractor",   Prompts.Grid);
    }
    public async Task<ExtractionResult> ExtractFromImageAsync(PageImage page, CancellationToken ct) {
        var message = new ChatMessage(ChatRole.User, new AIContent[] {
            new TextContent($"This is page {page.PageNumber}. Extract per the rules."),
            new DataContent(page.PngBytes, "image/png") });
        var response = await _vision.RunAsync<ExtractionResult>(
            message, serializerOptions: Json.Options, cancellationToken: ct);
        return response.Result; }
    public async Task<ExtractionResult> ExtractFromTextAsync(string markdown, int? pageHint, CancellationToken ct) {
        var response = await _text.RunAsync<ExtractionResult>(
            markdown, serializerOptions: Json.Options, cancellationToken: ct);
        return response.Result; }
    public async Task<ExtractionResult> ExtractFromGridAsync(string sheetGridJson, CancellationToken ct) {
        var response = await _grid.RunAsync<ExtractionResult>(
            sheetGridJson, serializerOptions: Json.Options, cancellationToken: ct);
        return response.Result; } }
Verified by build (MAF 1.12.0 + Extensions.AI 10.6.0, .NET 10). Compiles as written. ChatClientAgentOptions has Name + ChatOptions but no Instructions (put it on ChatOptions.Instructions, alongside Temperature). RunAsync<T>(message, serializerOptions:, cancellationToken:) returns a response with .Result (use var). Passing Json.Options yields snake_case string enums matching this document. Records use init defaults (no required) for reliable structured-output deserialization. Requires a vision + structured-output model (e.g. Azure OpenAI gpt-4o).
GenCore timeout resilience — implemented (shared Core). The extractor now STREAMS every call (RunStreamingAsync + explicit ChatOptions.ResponseFormat schema, assembled before deserializing) to defeat gateway idle timeouts; total-duration caps are handled by bounding every request — page-per-call vision, a chunked text leg (MarkdownChunker, block-boundary splits, tables never broken, default 24k chars), and per-sheet grid calls with a MaxCellsPerSheet cap. Each call retries ×3 with backoff and a permanently failed page/chunk/sheet becomes a report warning instead of aborting the run. Vision + text legs run concurrently under a shared SemaphoreSlim (--max-parallel); payloads use compact JSON. Full detail: spec-telerik.html §9.1 — the Core implementation is identical for both engines.
OSS-engine specific: Docnet's DocLib.Instance is a process-wide singleton and PDFium is not thread-safe. With the legs now concurrent, the adapters must NOT dispose the instance per call and must serialize PDFium work through a shared lock (Pdfium.Lock — implemented in LibreOfficeRenderer.Rasterize and OpenXmlTextExtractor.ExtractPdfText).

10.1 Renderer — docx/pdf/xlsx → PNG (Gotenberg + Docnet.Core + SkiaSharp)

// RfpExtractor.LibreOffice/LibreOfficeRenderer.cs
using System.Net.Http.Headers; using System.Runtime.InteropServices;
using Docnet.Core; using Docnet.Core.Converters; using Docnet.Core.Models; using SkiaSharp;

public sealed class LibreOfficeRenderer : IDocumentRenderer {
    private readonly HttpClient _http; private readonly string _gotenbergUrl;
    public LibreOfficeRenderer(HttpClient http, string gotenbergUrl) { _http = http; _gotenbergUrl = gotenbergUrl; }

    public async Task<IReadOnlyList<PageImage>> RenderToImagesAsync(string path, int dpi, CancellationToken ct) {
        // Gotenberg's LibreOffice route converts docx AND xlsx (and more) to PDF.
        byte[] pdf = path.EndsWith(".pdf", StringComparison.OrdinalIgnoreCase)
            ? await File.ReadAllBytesAsync(path, ct)
            : await ConvertViaGotenbergAsync(path, ct);
        return Rasterize(pdf, dpi); }

    private async Task<byte[]> ConvertViaGotenbergAsync(string path, CancellationToken ct) {
        var endpoint = _gotenbergUrl.TrimEnd('/') + "/forms/libreoffice/convert";
        using var form = new MultipartFormDataContent();
        var content = new ByteArrayContent(await File.ReadAllBytesAsync(path, ct));
        content.Headers.ContentType = new MediaTypeHeaderValue("application/octet-stream");
        form.Add(content, "files", Path.GetFileName(path));
        using var resp = await _http.PostAsync(endpoint, form, ct);
        resp.EnsureSuccessStatusCode();
        return await resp.Content.ReadAsByteArrayAsync(ct); }

    private static IReadOnlyList<PageImage> Rasterize(byte[] pdf, int dpi) {
        using var lib = DocLib.Instance;
        using var reader = lib.GetDocReader(pdf, new PageDimensions(dpi / 72.0));
        var pages = new List<PageImage>();
        for (int i = 0; i < reader.GetPageCount(); i++) {
            using var pr = reader.GetPageReader(i);
            int w = pr.GetPageWidth(), h = pr.GetPageHeight();
            byte[] bgra = pr.GetImage(new NaiveTransparencyRemover(255, 255, 255));
            using var bmp = new SKBitmap(w, h, SKColorType.Bgra8888, SKAlphaType.Premul);
            Marshal.Copy(bgra, 0, bmp.GetPixels(), bgra.Length);
            using var img = SKImage.FromBitmap(bmp);
            using var data = img.Encode(SKEncodedImageFormat.Png, 100);
            pages.Add(new PageImage(i + 1, data.ToArray())); }
        return pages; } }

10.2 Structured text — docx → Markdown + ground truth (Open XML SDK)

Upgraded: recursive, nested-table aware (field finding). Questionnaires often nest data-entry grids inside a big layout table; a flat walk loses them. The extractor now recurses: a table that contains tables is layout (short label cells → ## headings, cell content rendered recursively), leaf tables become markdown pipe tables and count toward the ground truth. This extractor is now used by both engines for docx text (Telerik's markdown export flattens nested tables).
// RfpExtractor.LibreOffice/OpenXmlTextExtractor.cs
using System.Text; using DocumentFormat.OpenXml.Packaging; using DocumentFormat.OpenXml.Wordprocessing;
using Docnet.Core; using Docnet.Core.Models;

public sealed class OpenXmlTextExtractor : IStructuredTextExtractor {
    public Task<StructuredDocument> ExtractAsync(string path, CancellationToken ct) {
        if (path.EndsWith(".pdf", StringComparison.OrdinalIgnoreCase)) return Task.FromResult(ExtractPdfText(path));
        using var doc = WordprocessingDocument.Open(path, false);
        var body = doc.MainDocumentPart!.Document.Body!;
        var sb = new StringBuilder(); var tables = new List<TableStructure>(); int tIdx = 0;
        foreach (var el in body.ChildElements) {
            if (el is Paragraph p) {
                var text = p.InnerText;
                if (string.IsNullOrWhiteSpace(text)) { sb.AppendLine(); continue; }
                var style = p.ParagraphProperties?.ParagraphStyleId?.Val?.Value ?? "";
                if (style.StartsWith("Heading", StringComparison.OrdinalIgnoreCase)) {
                    var level = int.TryParse(new string(style.Where(char.IsDigit).ToArray()), out var lv) ? lv : 1;
                    sb.AppendLine($"{new string('#', Math.Clamp(level, 1, 6))} {text}"); }
                else sb.AppendLine(text);
                sb.AppendLine(); }
            else if (el is Table tbl) {
                var rows = tbl.Elements<TableRow>().ToList();
                int cols = rows.Count == 0 ? 0 : rows.Max(r => r.Elements<TableCell>().Count());
                tables.Add(new TableStructure(tIdx++, rows.Count, cols, 0));
                for (int r = 0; r < rows.Count; r++) {
                    var cells = rows[r].Elements<TableCell>().Select(c => c.InnerText.Replace("|", "\\|").Trim()).ToList();
                    while (cells.Count < cols) cells.Add("");
                    sb.AppendLine("| " + string.Join(" | ", cells) + " |");
                    if (r == 0) sb.AppendLine("| " + string.Join(" | ", Enumerable.Repeat("---", cols)) + " |"); }
                sb.AppendLine(); } }
        return Task.FromResult(new StructuredDocument(sb.ToString(), tables)); }

    private static StructuredDocument ExtractPdfText(string path) {
        using var lib = DocLib.Instance;
        using var reader = lib.GetDocReader(path, new PageDimensions(1.0));
        var sb = new StringBuilder();
        for (int i = 0; i < reader.GetPageCount(); i++) {
            using var pr = reader.GetPageReader(i); sb.AppendLine(pr.GetText()); sb.AppendLine(); }
        return new StructuredDocument(sb.ToString(), Array.Empty<TableStructure>()); } }

10.3 Spreadsheet — xlsx → exact cell grid (ClosedXML)

// RfpExtractor.LibreOffice/ClosedXmlSpreadsheetExtractor.cs
using ClosedXML.Excel;

public sealed class ClosedXmlSpreadsheetExtractor : ISpreadsheetExtractor {
    public Task<WorkbookGrid> ExtractAsync(string path, CancellationToken ct) {
        using var wb = new XLWorkbook(path);
        var sheets = new List<SheetGrid>(); int si = 0;
        foreach (var ws in wb.Worksheets) {
            var used = ws.RangeUsed();                          // IXLRange, null if the sheet is empty
            var cells = new List<GridCell>();
            if (used != null) {
                var a = used.RangeAddress;
                for (int r = a.FirstAddress.RowNumber; r <= a.LastAddress.RowNumber; r++)      // 1-based
                for (int c = a.FirstAddress.ColumnNumber; c <= a.LastAddress.ColumnNumber; c++) {
                    var cell = ws.Cell(r, c);
                    string text = cell.GetString().Trim();
                    cells.Add(new GridCell($"{cell.Address.ColumnLetter}{cell.Address.RowNumber}",
                        r - 1, c - 1, text, cell.IsEmpty())); } }
            sheets.Add(new SheetGrid(ws.Name, si++, cells)); }
        return Task.FromResult(new WorkbookGrid(sheets)); } }
Merged-cell nuance. ClosedXML reports a merged region's value on its anchor cell; other cells in the merge read empty. If your questionnaires merge header cells heavily, expand ws.MergedRanges and repeat the anchor's value across the region before building the grid, so header labels line up with the answer cells beneath them.

14. Acceptance Criteria

  1. docx, pdf, and xlsx inputs each produce the four JSON outputs.
  2. Word/PDF data-entry cells and Excel answer cells each become a natural-language question; Excel questions carry binding.kind="cell" with correct sheet + address.
  3. The 1:1 invariant holds (or errors surface in report warnings).
  4. Every question carries found_by/confidence/needs_review; single-source items are in the review queue.
  5. MAF structured output deserializes into ExtractionResult with no manual parsing.
  6. Router selects spreadsheet pipeline for .xlsx; scanned PDFs degrade to vision-only.
  7. Unit tests: invariant, reconciliation dedup (text + cell), undercount warning.
