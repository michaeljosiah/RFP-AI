RFP Question Extractor Telerik + Microsoft Agent Framework

A .NET console application that extracts every question from Word, PDF and Excel questionnaires into two linked JSON documents — a structure schema and a flat answerable question list. LLM calls go through Microsoft Agent Framework (model-agnostic, structured output). Rendering and document parsing use Telerik Document Processing. Core logic is isolated behind interfaces so it can later be exposed as an LLM tool.

Who this is for. Written so a junior developer — or a less capable coding agent — can build the whole thing in one pass. Every model, interface, prompt, and the critical implementation code is given in full. Mechanical steps are marked // TODO.
Implementation status. This spec has been built and verified. The full solution (Core + Telerik + CLI + xUnit tests) compiles with 0 warnings on .NET 10; 14 tests pass — 10 Core unit tests (invariant, reconciliation, page-stitching) plus a real-document corpus (tests/…/TestData/) that runs the Telerik adapters over real questionnaires (FUND DUE DILIGENCE.docx → 20k markdown / 1 table; 1. EQDP RFP Questionnaire.docx → 28k markdown / 13 tables) asserting clean markdown + table detection + PNG rendering. The Telerik engine was also run against the M&G questionnaire (docx → 8.2k-char markdown + 1 table + 3 rendered pages) and a test workbook (xlsx → 6 label + 6 empty answer cells with exact addresses). The LLM legs compile against MAF 1.12.0 but need an Azure OpenAI (or other IChatClient) endpoint to run end-to-end. Deviations found during the build are folded into the sections below and flagged "verified by build".

1. Overview & Goals

Investment-management RFP / due-diligence questionnaires arrive as Word (.docx), PDF, or Excel (.xlsx). We turn them into a concrete, answerable question list and record exactly where each answer belongs so a downstream tool can write answers back.

  • Word / PDF — a dual pipeline: a vision leg (render each page → image → agent) and a structured-text leg (Telerik Markdown export → agent), then reconciliation.
  • Excel — a grid-first pipeline: read the cells directly (exact addresses) → agent classifies labels vs answer cells; optional vision cross-check. Excel answers get exact Sheet!Cell bindings, so no annotation step is needed for fill-back.
  • Reconciliation — merge legs, tag each question with found_by + confidence, route single-source items to a review queue.

Why MAF lives in Core. Microsoft.Agents.AI and Microsoft.Extensions.AI are provider-agnostic abstraction packages (no vendor lock-in), so they belong in Core. The concrete model provider (Azure OpenAI, OpenAI, Anthropic, Foundry) is chosen in the CLI by constructing an IChatClient and passing it in. Swapping model providers never touches Core.

4.3 The 1:1 invariant

Invariant. The set of answer_target ids in document_schema (every non-table item's answer_target + every table cell's) equals the set referenced by questions. No orphans, no duplicates. Validate after every extraction and after reconciliation.
// RfpExtractor.Core/Validation/InvariantValidator.cs
public static class InvariantValidator {
    public static IReadOnlyList<string> Validate(ExtractionResult r) {
        var errors = new List<string>(); var schema = new HashSet<string>();
        foreach (var s in r.DocumentSchema.Sections)
        foreach (var it in s.Items) {
            if (it.Type == ItemType.Table && it.Table is not null)
                foreach (var c in it.Table.Cells) Add(schema, c.AnswerTarget, errors);
            else if (it.AnswerTarget is { } t) Add(schema, t, errors); }
        var q = new HashSet<string>();
        foreach (var it in r.Questions) {
            if (!q.Add(it.AnswerTarget)) errors.Add($"Duplicate question target: {it.AnswerTarget}");
            if (!schema.Contains(it.AnswerTarget)) errors.Add($"Question {it.QuestionId} -> missing target {it.AnswerTarget}"); }
        foreach (var t in schema) if (!q.Contains(t)) errors.Add($"Schema target {t} has no question");
        return errors;
        static void Add(HashSet<string> set, string t, List<string> e){ if(!set.Add(t)) e.Add($"Duplicate schema target: {t}"); } } }

5. Prompts

String constants in Prompts.cs. Each becomes an agent's instructions (§9). The response is forced to the schema by structured output, so the model returns only the object.

5.1 Vision

Vision-mode precision (field finding — image-heavy forms). The vision prompt adds transcription rules: copy printed identifiers (regulation refs like 2009/65/EC, dates, article/part numbers, ISINs, %, currency) exactly — never "correct" a digit or spelling — and set unreadable:true rather than guess. On the Fund-DD form the vision leg had misread 2009/65/EC → 2006/95/EC and March 26th → 25th; those surfaced correctly as low-confidence review items, and the rule reduces such OCR substitutions. Cross-source reconciliation (§8) merges the rest.
You are a precise document-extraction engine for RFP/questionnaire automation.
You are given ONE image of a single page. Return a single object with "document_schema"
and "questions" as structured output. Do not add commentary.

CORE
- Mint a unique "answer_target" ("AT-0001", "AT-0002", ... in reading order) for EVERY
  answerable location: each open question, each document/data request line, each yes/no,
  and each fillable data-entry table cell.
- INVARIANT: every answer_target appears exactly once in document_schema AND exactly once
  in questions. No orphans, no duplicates.

RULES
1. VERBATIM in the schema; the questions list may NORMALISE phrasing but keep the original
   in "verbatim_source". Never answer anything.
2. TABLES: "layout" tables (label in one column, questions in another) are NOT emitted as
   tables - extract their content as normal items and use label cells as section names.
   "data_entry" grids ARE emitted: column_headers, row_headers, intro_verbatim, one cell
   per fillable cell (own answer_target,row,column,answer_type) AND one question per cell,
   phrased standalone from context+column+row. e.g. "What was the firm's AUM 1 year ago?"
3. MULTI-PART bundled into one answer box -> one question + "sub_questions"; separate boxes
   -> separate targets.
4. answer_type: text|long_text|number|currency|percentage|date|yes_no|document_upload.
5. SECTIONS from headings/bands; capture section+subsection+page; slug ids; reading order.
6. "truncated": last item cut off at the page edge; "unreadable": partly illegible (+best
   guess). Exclude headers/footers/page-numbers/watermarks/legal boilerplate; put
   document-identifying fields in metadata.

5.2 Structured text

You are a precise document-extraction engine. You are given the FULL document as Markdown
(tables preserved; headings as '#'). Return a single object with "document_schema" and
"questions" as structured output.

[... identical CORE and RULES 1-6 as the Vision prompt ...]

TEXT MODE: markdown tables are literal - header row = column_headers; leftmost body column
= row header; every EMPTY body cell is a fillable data_entry cell (own answer_target +
question). A body cell containing text is context, not an answer cell.

5.3 Reconciliation

You merge two independent extractions of the SAME questionnaire. Input JSON has "primary"
(questions[]) and "secondary" (questions[]). Return a single "questions" array as
structured output.

1. Same question if normalised meaning matches (ignore casing/whitespace/rewording) OR they
   target the same cell (same row+column under the same section). Merge duplicates.
2. On merge PREFER primary's verbatim_source + answer_target; pick the cleaner question_text;
   union sub_questions.
3. found_by = "both" if in both; else the single source ("vision"/"text").
4. confidence = high if both; medium if single-source table cell; low otherwise.
5. needs_review = true whenever found_by != both. Never drop single-source questions.
6. Preserve every field; keep answer_target stable. Never invent questions.

5.4 Excel grid

You are a precise spreadsheet-extraction engine for RFP/questionnaire automation.
You are given ONE worksheet as JSON: { "sheet": <name>, "cells": [ {address, text} ... ] },
where answer cells have text = "". Return a single object with "document_schema" and
"questions" as structured output.

1. Classify LABEL cells (question text, row/column headers, section banners) vs ANSWER cells
   (empty cells the respondent fills, inside the used range under a header and/or beside a
   row label).
2. For EACH answer cell emit: a schema entry (a data_entry table cell when it sits in a
   header x row grid, else an open_question item) with a unique answer_target, AND one
   question. question_text is a fluent standalone question from section + column header +
   row label. e.g. column "Firm AUM", row "1 Year ago" -> "What was the firm's AUM 1 year ago?"
3. Put binding = { "kind":"cell", "sheet":<name>, "address":<A1, e.g. B3> } on the question
   so the answer writes straight back to that cell. Also set schema_ref.row/column to the
   label texts.
4. answer_type inferred from header wording (currency/number/percentage/date/text).
5. verbatim_source = exact label texts; question_text may be normalised.
6. INVARIANT: one answer_target per answer cell, referenced by exactly one question. Ignore
   styling-only or out-of-range cells.

6. Document Pipeline (Word / PDF)

// RfpExtractor.Core/Pipeline/DocumentPipeline.cs
public enum Strategy { Vision, Text, Both }
public sealed record ExtractionOptions(Strategy Strategy = Strategy.Both, int Dpi = 200, int MaxParallel = 4);

public sealed class DocumentPipeline {
    private readonly IDocumentRenderer _renderer; private readonly IStructuredTextExtractor _text;
    private readonly ILlmExtractor _llm; private readonly IReconciler _reconciler;
    public DocumentPipeline(IDocumentRenderer r, IStructuredTextExtractor t, ILlmExtractor l, IReconciler rec)
    { _renderer=r; _text=t; _llm=l; _reconciler=rec; }

    public async Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        ExtractionResult vision = new();
        if (opts.Strategy is Strategy.Vision or Strategy.Both) {
            var pages = await _renderer.RenderToImagesAsync(path, opts.Dpi, ct);
            var sem = new SemaphoreSlim(opts.MaxParallel);
            var per = await Task.WhenAll(pages.Select(async p => {
                await sem.WaitAsync(ct);
                try { return await _llm.ExtractFromImageAsync(p, ct); } finally { sem.Release(); } }));
            vision = ResultMerger.StitchPages(per); }

        ExtractionResult text = new(); StructuredDocument? structured = null;
        if (opts.Strategy is Strategy.Text or Strategy.Both) {
            structured = await _text.ExtractAsync(path, ct);
            if (!string.IsNullOrWhiteSpace(structured.Markdown))
                text = await _llm.ExtractFromTextAsync(structured.Markdown, null, ct); }

        var gt = structured?.Tables ?? Array.Empty<TableStructure>();
        var result = opts.Strategy switch {
            Strategy.Vision => ReconciledResult.FromSingle(vision),
            Strategy.Text   => ReconciledResult.FromSingle(text),
            _               => _reconciler.Reconcile(primary: text, secondary: vision, gt) };  // text is authoritative

        foreach (var e in InvariantValidator.Validate(result.Merged)) result.Report.Warnings.Add(e);
        return result; } }
ResultMerger.StitchPages concatenates per-page schemas, renumbers AT-####/Q### continuously, and joins any item flagged truncated:true with the first item of the next page. Implement as a straight loop (mechanical).

7. Excel Pipeline

Excel is a natively addressable grid, so the grid leg is authoritative (exact cell addresses, no OCR) and vision is an optional cross-check. Each sheet is treated like a page. Fill-back bindings are exact (Sheet!Cell), so Excel needs no annotation step.

// RfpExtractor.Core/Pipeline/SpreadsheetPipeline.cs
public sealed class SpreadsheetPipeline {
    private readonly ISpreadsheetExtractor _grid; private readonly IDocumentRenderer _renderer;
    private readonly ILlmExtractor _llm; private readonly IReconciler _reconciler;
    public SpreadsheetPipeline(ISpreadsheetExtractor g, IDocumentRenderer r, ILlmExtractor l, IReconciler rec)
    { _grid=g; _renderer=r; _llm=l; _reconciler=rec; }

    public async Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        // --- primary: grid classification, one agent call per sheet ---
        var wb = await _grid.ExtractAsync(path, ct);
        var perSheet = new List<ExtractionResult>();
        foreach (var sheet in wb.Sheets) {
            var payload = System.Text.Json.JsonSerializer.Serialize(new {
                sheet = sheet.Name,
                cells = sheet.Cells.Select(c => new { c.Address, c.Text }) }, Json.Options);
            perSheet.Add(await _llm.ExtractFromGridAsync(payload, ct)); }
        var gridResult = ResultMerger.StitchPages(perSheet.ToArray());

        if (opts.Strategy != Strategy.Both) {
            var single = ReconciledResult.FromSingle(gridResult);
            foreach (var e in InvariantValidator.Validate(single.Merged)) single.Report.Warnings.Add(e);
            return single; }

        // --- optional secondary: vision cross-check (xlsx -> pdf -> image) ---
        var pages = await _renderer.RenderToImagesAsync(path, opts.Dpi, ct);
        var per = await Task.WhenAll(pages.Select(p => _llm.ExtractFromImageAsync(p, ct)));
        var vision = ResultMerger.StitchPages(per);

        var result = _reconciler.Reconcile(primary: gridResult, secondary: vision, Array.Empty<TableStructure>());
        foreach (var e in InvariantValidator.Validate(result.Merged)) result.Report.Warnings.Add(e);
        return result; } }

// RfpExtractor.Core/Pipeline/PipelineRouter.cs
public sealed class PipelineRouter {
    private readonly DocumentPipeline _doc; private readonly SpreadsheetPipeline _sheet;
    public PipelineRouter(DocumentPipeline doc, SpreadsheetPipeline sheet) { _doc=doc; _sheet=sheet; }
    public Task<ReconciledResult> RunAsync(string path, ExtractionOptions opts, CancellationToken ct) {
        var ext = Path.GetExtension(path).ToLowerInvariant();
        return ext is ".xlsx" or ".xlsm" or ".xls"
            ? _sheet.RunAsync(path, opts, ct) : _doc.RunAsync(path, opts, ct); } }
Default strategy for Excel is grid-only (Strategy.Text maps to grid). Pass --strategy=both to add the vision cross-check for confidence. For very large sheets, cap the cells per agent call (e.g. 4,000) and log the truncation.

8. Reconciliation & Confidence

v2 — rebuilt after field testing (M&G questionnaire run). The first deterministic reconciler had three defects the real run exposed; the implementation now works as follows:
• One-to-one ordered matching: FIFO queues per key over unmatched primary questions; each secondary question consumes at most one. Documents legitimately repeat questions across sections (per-fund repeats) — those are distinct answer slots, so same-leg repeats are never collapsed and reading-order pairing aligns them.
• Verbatim-first keys: table cells match on row+column; body questions match on normalized verbatim_source (the printed text — stable across legs), then question_text. Keying on question_text alone fails because each leg may rephrase it.
• LLM fuzzy pass (AgentFuzzyMatcher, prompt Prompts.FuzzyMatch, default ON, --no-fuzzy to disable, 120-per-side cap): pairs the paraphrase leftovers; failures degrade to the deterministic result with a warning.
• Renumber + graft: the two legs number AT-#### independently, so a plain union collides; after matching, ALL targets are renumbered into one namespace and secondary-only items (e.g. table cells only vision can see) are grafted into the merged schema under vision-* sections — the 1:1 invariant holds on the merged output.
• Granularity rules added to the extraction prompts: document-request lists = one question per listed document; multi-part text sharing one answer box = exactly one question with sub_questions.

// RfpExtractor.Core/Reconciliation/TextNormalizer.cs
public static class TextNormalizer {
    public static string Key(string s) => System.Text.RegularExpressions.Regex.Replace(
        (s ?? "").ToLowerInvariant().Trim(), @"[\s\p{P}]+", " ").Trim();
    public static string CellKey(SchemaRef r) => $"{r.SectionId}|{r.Row?.ToLowerInvariant()}|{r.Column?.ToLowerInvariant()}"; }

// RfpExtractor.Core/Reconciliation/Reconciler.cs   (primary = authoritative leg)
public sealed class Reconciler : IReconciler {
    public ReconciledResult Reconcile(ExtractionResult primary, ExtractionResult secondary,
                                      IReadOnlyList<TableStructure> groundTruth) {
        var merged = new Dictionary<string, Question>();
        var report = new ReconciliationReport { PrimaryCount = primary.Questions.Count, SecondaryCount = secondary.Questions.Count };
        void Ingest(IEnumerable<Question> qs, bool isPrimary) {
            foreach (var q in qs) {
                var key = q.Source == QuestionSource.TableCell ? "cell:" + TextNormalizer.CellKey(q.SchemaRef)
                                                               : "txt:"  + TextNormalizer.Key(q.QuestionText);
                if (merged.TryGetValue(key, out var e))
                    merged[key] = e with { FoundBy = FoundBy.Both, Confidence = Confidence.High, NeedsReview = false,
                        SubQuestions = e.SubQuestions.Union(q.SubQuestions).ToList() };
                else
                    merged[key] = q with {
                        FoundBy = isPrimary ? FoundBy.Text : FoundBy.Vision,
                        Confidence = q.Source == QuestionSource.TableCell ? Confidence.Medium : Confidence.Low,
                        NeedsReview = true }; } }
        Ingest(primary.Questions, true);        // primary first so its verbatim/target/binding win
        Ingest(secondary.Questions, false);

        var cellCount = merged.Values.Count(q => q.Source == QuestionSource.TableCell);
        var expected = groundTruth.Sum(t => Math.Max(0, (t.Rows - 1) * (t.Columns - 1)));
        if (expected > 0 && cellCount < expected)
            report.Warnings.Add($"Table cell undercount: found {cellCount}, grid implies ~{expected}.");

        var all = merged.Values.ToList();
        report.MergedCount = all.Count; report.AgreedCount = all.Count(q => q.FoundBy == FoundBy.Both);
        report.PrimaryOnlyCount = all.Count(q => q.FoundBy == FoundBy.Text);
        report.SecondaryOnlyCount = all.Count(q => q.FoundBy == FoundBy.Vision);
        var schema = primary.DocumentSchema.Sections.Count > 0 ? primary.DocumentSchema : secondary.DocumentSchema;
        return new ReconciledResult {
            Merged = new ExtractionResult { DocumentSchema = schema, Questions = all },
            ReviewQueue = all.Where(q => q.NeedsReview).ToList(), Report = report }; } }
Optional LLM fuzzy pass. Send only the needs_review items through the reconciliation agent (§5.3) to catch near-duplicates the normalizer missed. Keep behind a --fuzzy-reconcile flag.

9. LLM via Microsoft Agent Framework

All LLM work goes through MAF agents. Core references only the abstraction packages; the model provider is injected as an IChatClient. One AgentLlmExtractor builds three agents (one per prompt) and uses structured output (RunAsync<ExtractionResult>) so results deserialize straight into our records. Vision is a DataContent image block on the message.

<!-- RfpExtractor.Core.csproj (abstraction packages only) -->
<PackageReference Include="Microsoft.Agents.AI" Version="1.*-*" />   <!-- resolves 1.12.0 (stable) -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" /> <!-- MAF 1.12 requires >= 10.6.0 -->
// RfpExtractor.Core/Llm/AgentLlmExtractor.cs
using Microsoft.Agents.AI;
using Microsoft.Extensions.AI;
using RfpExtractor.Core.Abstractions;
using RfpExtractor.Core.Models;
using RfpExtractor.Core.Prompts;

public sealed class AgentLlmExtractor : ILlmExtractor
{
    private readonly AIAgent _vision, _text, _grid;

    public AgentLlmExtractor(IChatClient chat)
    {
        // Structured output via an EXPLICIT JSON schema on ChatOptions.ResponseFormat so responses
        // can be streamed (RunAsync<T> hides the schema but cannot stream). Schema is derived from
        // the ExtractionResult records with snake_case string enums via Json.Options.
        var schema = AIJsonUtilities.CreateJsonSchema(typeof(ExtractionResult), serializerOptions: Json.Options);
        var responseFormat = ChatResponseFormat.ForJsonSchema(schema, "extraction_result",
            "Document schema and flat question list extracted from a questionnaire.");

        // One agent per prompt. Temperature 0 for determinism.
        // NOTE (verified against MAF 1.12.0): ChatClientAgentOptions has NO Instructions property -
        // set instructions on ChatOptions.Instructions instead.
        AIAgent Make(string name, string instructions) => new ChatClientAgent(chat,
            new ChatClientAgentOptions {
                Name = name,
                ChatOptions = new ChatOptions {
                    Instructions = instructions, Temperature = 0f, ResponseFormat = responseFormat } });

        _vision = Make("vision-extractor", Prompts.Vision);
        _text   = Make("text-extractor",   Prompts.Text);
        _grid   = Make("grid-extractor",   Prompts.Grid);
    }

    public Task<ExtractionResult> ExtractFromImageAsync(PageImage page, CancellationToken ct)
    {
        var message = new ChatMessage(ChatRole.User, new AIContent[] {
            new TextContent($"This is page {page.PageNumber}. Extract per the rules."),
            new DataContent(page.PngBytes, "image/png") });
        return RunStreamingAsync(_vision, message, ct);
    }

    public Task<ExtractionResult> ExtractFromTextAsync(string markdown, int? pageHint, CancellationToken ct)
        => RunStreamingAsync(_text, new ChatMessage(ChatRole.User, markdown), ct);

    public Task<ExtractionResult> ExtractFromGridAsync(string sheetGridJson, CancellationToken ct)
        => RunStreamingAsync(_grid, new ChatMessage(ChatRole.User, sheetGridJson), ct);

    // STREAM the response and assemble before deserializing. Streaming buys no latency on the
    // final structured result, but keeps bytes flowing on the wire - which defeats gateway IDLE
    // timeouts (GenCore cuts silent long-running requests). Requires the explicit ResponseFormat
    // schema above (the streaming-compatible equivalent of RunAsync<T>).
    private static async Task<ExtractionResult> RunStreamingAsync(AIAgent agent, ChatMessage message, CancellationToken ct)
    {
        var sb = new StringBuilder();
        await foreach (var update in agent.RunStreamingAsync(message, cancellationToken: ct))
            sb.Append(update.Text);
        var json = sb.ToString();
        if (string.IsNullOrWhiteSpace(json))
            throw new InvalidOperationException("Model returned an empty response.");   // triggers pipeline retry
        return JsonSerializer.Deserialize<ExtractionResult>(json, Json.Options)
            ?? throw new InvalidOperationException("Model response deserialized to null.");
    }
}
Verified by build (MAF 1.12.0 + Extensions.AI 10.6.0, .NET 10). The code above compiles as written. Confirmed: (1) new ChatClientAgent(chat, ChatClientAgentOptions) is the ctor; (2) ChatClientAgentOptions has Name + ChatOptions but no Instructions — instructions go on ChatOptions.Instructions; (3) ChatOptions exposes Instructions and Temperature; (4) RunAsync<T>(message, serializerOptions:, cancellationToken:) returns a response with .Result (use var — the concrete type name varies by build); (5) vision works via a ChatMessage holding TextContent + DataContent(bytes,"image/png").
Model + serialization. Structured output + vision needs a capable model (e.g. Azure OpenAI gpt-4o). The explicit ResponseFormat schema is built with Json.Options so the wire format uses snake_case string enums, matching this document. The domain records use init defaults (no required modifier) so structured-output deserialization populates them reliably.

9.1 GenCore timeout resilience implemented

GenCore (the enterprise gateway) is notorious for cutting long-running requests, and typical documents run 30–50 pages. Two failure modes need two different mitigations, plus supporting measures — all implemented:

MeasureDefeatsWhere
Bound every request — one page per vision call; the text leg is CHUNKED by MarkdownChunker (block-boundary splits, tables never broken, default 24k chars ≈ 6k tokens, ExtractionOptions.TextChunkChars); Excel is one sheet per call with a MaxCellsPerSheet cap (default 4,000, truncation warned). Total-duration caps — no single call generates an unbounded response. Also parallelises the text leg (chunks fan out like pages). Pipelines.cs, MarkdownChunker.cs
Streaming transport — every agent call uses RunStreamingAsync, assembling updates before deserializing. Idle timeouts — bytes flow continuously, the connection never looks silent. AgentLlmExtractor.RunStreamingAsync
Per-call retry + partial-failure tolerance — each LLM call retries up to 3 attempts with linear backoff (RetryDelay); a permanently failed page/chunk/sheet yields an empty result plus a reconciliation_report.json warning instead of aborting the run (60+ calls per large document makes a transient blip a statistical certainty). Transient gateway errors surviving the other measures. Resilience.SafeExtractAsync
Concurrent legs + tunable parallelism — vision and text legs run under Task.WhenAll sharing one SemaphoreSlim(MaxParallel); Excel sheets fan out the same way; --max-parallel tunes to the GenCore rate quota. Progress callbacks (ExtractionOptions.OnProgress) keep multi-minute runs observable. Wall-clock (= max(legs), not sum) and "is it hung?" ambiguity. DocumentPipeline, SpreadsheetPipeline, CLI
Token-lean payloads — LLM payloads serialize with Json.Compact (no indentation); sheet payloads send non-empty cells as objects and empty cells (the answer candidates) as a bare address array. Cost + request duration (fewer input tokens → faster calls). SpreadsheetPipeline.BuildSheetPayload, Json.cs
Open question for the GenCore team. Is the gateway timeout idle-based or a hard total-duration cap, and what is the number? The design covers both, but the answer calibrates TextChunkChars (smaller chunks → shorter calls) and --max-parallel (RPM/TPM quota).

// RfpExtractor.Telerik/TelerikRenderer.cs
using Telerik.Documents.Fixed.FormatProviders.Image.Skia;
using Telerik.Windows.Documents.Fixed.Model;
using DocxProvider   = Telerik.Windows.Documents.Flow.FormatProviders.Docx.DocxFormatProvider;
using FlowToPdf      = Telerik.Windows.Documents.Flow.FormatProviders.Pdf.PdfFormatProvider;
using FixedPdf       = Telerik.Windows.Documents.Fixed.FormatProviders.Pdf.PdfFormatProvider;
using XlsxProvider   = Telerik.Windows.Documents.Spreadsheet.FormatProviders.OpenXml.Xlsx.XlsxFormatProvider;
using SheetToPdf     = Telerik.Windows.Documents.Spreadsheet.FormatProviders.Pdf.PdfFormatProvider;

public sealed class TelerikRenderer : IDocumentRenderer {
    public Task<IReadOnlyList<PageImage>> RenderToImagesAsync(string path, int dpi, CancellationToken ct) {
        var timeout = TimeSpan.FromSeconds(120);
        var ext = Path.GetExtension(path).ToLowerInvariant();
        byte[] pdf;
        if (ext == ".pdf") pdf = File.ReadAllBytes(path);
        else if (ext is ".xlsx" or ".xlsm" or ".xls") {
            using var fs = File.OpenRead(path);
            var wb = new XlsxProvider().Import(fs, timeout);
            using var ms = new MemoryStream(); new SheetToPdf().Export(wb, ms, timeout); pdf = ms.ToArray();
        } else {
            using var fs = File.OpenRead(path);
            var flow = new DocxProvider().Import(fs, timeout);
            pdf = new FlowToPdf().Export(flow, timeout);
        }
        RadFixedDocument fixedDoc = new FixedPdf().Import(pdf, timeout);
        var provider = new SkiaImageFormatProvider();
        provider.ExportSettings.ScaleFactor = dpi / 72.0;
        var pages = new List<PageImage>();
        for (int i = 0; i < fixedDoc.Pages.Count; i++)
            pages.Add(new PageImage(i + 1, provider.Export(fixedDoc.Pages[i], timeout)));
        return Task.FromResult<IReadOnlyList<PageImage>>(pages);
    } }

10.2 Structured text — docx → Markdown + table ground truth

Superseded for docx (field finding). Telerik's MarkdownFormatProvider flattens tables nested inside layout tables — on the M&G questionnaire the nested AUM/team grids were lost and the text leg emitted zero table cells. Both engines now use the recursive Open XML SDK extractor (OpenXmlTextExtractor, MIT): tables containing tables are treated as layout (label cells → ## headings, content recursed), leaf tables become real markdown pipe tables and count toward the ground truth. On the same document: 1 → 3 ground-truth tables, nested grids present with empty answer cells. The old TelerikTextExtractor has been removed (dead code once both engines switched).
// RfpExtractor.Telerik/TelerikTextExtractor.cs
using Telerik.Windows.Documents.Flow.Model;
using MarkdownProvider = Telerik.Windows.Documents.Flow.FormatProviders.Md.MarkdownFormatProvider;
using DocxProvider     = Telerik.Windows.Documents.Flow.FormatProviders.Docx.DocxFormatProvider;

public sealed class TelerikTextExtractor : IStructuredTextExtractor {
    public Task<StructuredDocument> ExtractAsync(string path, CancellationToken ct) {
        if (path.EndsWith(".pdf", StringComparison.OrdinalIgnoreCase))
            return Task.FromResult(new StructuredDocument("", Array.Empty<TableStructure>())); // vision-only for PDF
        var timeout = TimeSpan.FromSeconds(120);
        RadFlowDocument flow;
        using (var fs = File.OpenRead(path)) flow = new DocxProvider().Import(fs, timeout);
        string markdown = new MarkdownProvider().Export(flow, timeout);
        var tables = new List<TableStructure>(); int idx = 0;
        foreach (var section in flow.Sections)
        foreach (var block in section.Blocks)
            if (block is Table t) {
                int rows = t.Rows.Count;
                int cols = rows == 0 ? 0 : t.Rows.Max(r => r.Cells.Count);
                tables.Add(new TableStructure(idx++, rows, cols, 0)); }
        return Task.FromResult(new StructuredDocument(markdown, tables));
    } }

10.3 Spreadsheet — xlsx → exact cell grid

// RfpExtractor.Telerik/TelerikSpreadsheetExtractor.cs
using Telerik.Windows.Documents.Spreadsheet.Model;
using XlsxProvider = Telerik.Windows.Documents.Spreadsheet.FormatProviders.OpenXml.Xlsx.XlsxFormatProvider;

public sealed class TelerikSpreadsheetExtractor : ISpreadsheetExtractor {
    public Task<WorkbookGrid> ExtractAsync(string path, CancellationToken ct) {
        Workbook wb;
        using (var fs = File.OpenRead(path)) wb = new XlsxProvider().Import(fs, TimeSpan.FromSeconds(120));
        var sheets = new List<SheetGrid>(); int si = 0;
        foreach (var ws in wb.Worksheets) {
            var used = ws.UsedCellRange;                        // CellRange (may be null for empty sheet)
            var cells = new List<GridCell>();
            if (used != null) {
                for (int r = used.FromIndex.RowIndex; r <= used.ToIndex.RowIndex; r++)
                for (int c = used.FromIndex.ColumnIndex; c <= used.ToIndex.ColumnIndex; c++) {
                    CellSelection cell = ws.Cells[r, c];
                    string text = (cell.GetValue().Value?.RawValue ?? "").Trim();
                    cells.Add(new GridCell(A1(r, c), r, c, text, string.IsNullOrWhiteSpace(text))); } }
            sheets.Add(new SheetGrid(ws.Name, si++, cells)); }
        return Task.FromResult(new WorkbookGrid(sheets));
    }
    // 0-based (row,col) -> A1 (e.g. 0,0 -> "A1")
    private static string A1(int row, int col) {
        string s = ""; int c = col + 1;
        while (c > 0) { int m = (c - 1) % 26; s = (char)('A' + m) + s; c = (c - 1) / 26; }
        return $"{s}{row + 1}"; } }
Cell value note. cell.GetValue().Value.RawValue returns the underlying literal (for formula cells it is the formula). For display-formatted output use ICellValue.GetResultValueAsString(cell.GetFormat().Value) instead. For questionnaire label cells (text literals) RawValue is correct and simplest.

14. Acceptance Criteria

  1. docx, pdf, and xlsx inputs each produce document_schema.json, questions.json, review_queue.json, reconciliation_report.json.
  2. Word/PDF data-entry cells and Excel answer cells each become a natural-language question; Excel questions carry binding.kind="cell" with the correct sheet + address.
  3. The 1:1 invariant holds on the merged result (or errors surface in the report warnings).
  4. Every question carries found_by/confidence/needs_review; single-source items are in the review queue.
  5. MAF structured output deserializes into ExtractionResult with no manual JSON parsing.
  6. Router selects the spreadsheet pipeline for .xlsx and the document pipeline otherwise.
  7. Unit tests: invariant (pass + failure modes), reconciliation dedup (text + cell), undercount warning.
